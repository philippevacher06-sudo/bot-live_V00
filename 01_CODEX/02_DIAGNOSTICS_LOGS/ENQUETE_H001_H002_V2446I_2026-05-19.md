## 5. Blocs V2446I a regler - inventaire par severite

Les references de lignes sont indicatives et basees sur le commit `969bbd7`.

### Bloquant

- B1 - TP progressif non applique.
  Dans `close_winning_basket_if_needed` (runner, zone ~2495-2546), la decision
  de fermeture `total_upl >= tp` utilise le `tp` statique issu de
  `V2446I_BASKET_TAKE_PROFIT_EUR`. Le bloc commente "TP DYNAMIQUE" qui reassigne
  `tp` selon le nombre de legs se trouve dans la branche `if close_reason is None:`,
  donc apres la decision : il n'influence jamais une fermeture. Le panier ferme
  donc a TP fixe (1 EUR pour US500). Contredit les regles maitresses 43-45 et
  l'interdiction "remplacer le TP dynamique par un TP fixe simple".

- B2 - Renfort US100 repete.
  Le bloc "BIFURCATION L4 SUR US100" (`v2446_adverse_steps_patch.py`, ~447-484)
  se declenche sur `len(ps) >= 3`. `ps` ne compte que l'actif principal (US500) ;
  la position US100 ouverte n'incremente pas `ps`. Il n'existe aucune garde
  d'idempotence ni verification "L4 deja ouvert". Tant que le panier US500 reste
  a 3 legs, chaque iteration de boucle re-declenche un POST d'ordre US100.
  Contredit le scenario H001-S08 et le verrou "aucun renfort US100".

- B3 - Jambe hedge US100 orpheline.
  La position L4 US100 est ouverte par le patch adverse-step, mais le filtre
  `_v2446i_eth_positions` ne retient que l'actif principal. Consequence :
  l'UPL US100 est exclu de `total_upl` (donc du TP et du stop), et
  `_v2446i_close_all` ne ferme que les positions US500. La jambe US100 n'est
  donc ni comptee ni fermee par la cascade. Contredit H001-S09 / S10 et le
  verrou explicite du plan de patch "la fermeture globale ferme toutes les
  jambes". Le TP / stop "cumule US500 + US100" de la spec n'est pas honore.

### Eleve

- E1 - `V2446I_MAX_LEGS=4` pour US500. Le compteur de la cascade ne compte que
  US500. Avec 4, un 4e leg US500 est autorise alors que H-001 limite US500 a
  L1/L2/L3. Valeur attendue : 3 pour le panier principal.

- E2 - Recuperation fragile des headers API. Le hedge L4 obtient les en-tetes
  d'authentification via `inspect.currentframe()` puis `f_back.f_locals`. Toute
  evolution du chemin d'appel fait echouer silencieusement le hedge
  ("Headers API introuvables").

- E3 - Chemin d'execution du hedge non securise. Le hedge L4 fait un
  `requests.post` direct sur `/positions`, contournant `06G2_execution_secure`
  (reconciliation, confirmation). Les legs principaux et la jambe hedge ne
  passent donc pas par le meme chemin securise.

- E4 - PnL local aveugle BID/ASK. `BOT_PIVOT_00B_pnl_eur.py` calcule via un seul
  `current_price` et sa seule source de prix est le MID (`mid_for`). La regle
  BUY au BID / SELL a l'ASK (regle 49, AUD-011/012) n'est pas implementee dans
  ce module. La cascade utilise `broker_upl` (conforme), mais le chemin classique
  05x qui consomme ce module reste expose.

### Moyen

- M-a - Bloc "TP DYNAMIQUE" duplique a l'identique (deux fois de suite). Code
  mort, signe de patch empile (regle maitresse 10).
- M-b - Le filtre actif/epic compare une chaine majuscule exacte sans
  normalisation. Si l'epic broker reel differe de `US500` / `FR40`, la cascade
  ne voit aucune position et ne fait plus rien, silencieusement (lien AUD-002).
- M-c - L'adverse step est en pourcentage (`STEP_PCT=0.00044`) alors que la spec
  H-001 parle de 3 points absolus ; l'ecart effectif varie avec le niveau de prix.
- M-d - Le time-stop n'est arme que si `total_upl < 0`. Un panier gagnant n'est
  pas ferme a 7200 s, alors que la fixture H001-S13 attend une fermeture totale.
- M-e - Distance de stop du hedge codee en dur (prix +/- 150), non parametree.
- M-f - 411 fichiers `logs/` et `backups/` sont versionnes malgre `.gitignore`
  (ajoute apres coup, sans effet retroactif). Depot a 285 Mo.

### Faible / coherence documentaire

- F-a - Host broker incoherent : 6 occurrences `backend.capital.com` dans les
  fichiers `PATCH_V2447_*` contre 127 occurrences `backend-capital.com` ailleurs.
- F-b - Le protocole d'audit cite `00_COMMUN_SOURCE_DE_VERITE/`, dossier
  inexistant (reel : `00_COMMUN/`).
- F-c - Prefixes legacy `eth` / `_v2445` sur des fonctions devenues generiques :
  piege de lecture pour un futur intervenant.
- F-d - `V2446I_TIME_STOP_SEC=7200` cote US500 correspond bien a la spec H-001,
  mais entre en tension avec la regle maitresse 48 (14400). A harmoniser dans
  la doctrine (override per-strategie a documenter).
